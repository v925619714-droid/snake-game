## ⚖️ Вердикт
**Функционально приложение в хорошей форме** — большинство частых причин отказа уже закрыты
(privacy policy, удаление аккаунта, необязательный логин, нет ATT/IDFA, нет нарушений IAP,
дифференцированный геймплей). **Реальные стопперы — не правила, а:**
1. ❌ **Нет активного Apple Developer Program** ($99/год, T17) — без него нельзя ни собрать prod-билд, ни отправить. Главный блокер.
2. ⚠️ **Письмо с кодом входа доходит только на owner-email** (Resend без верифиц. домена) — если ревьюер проверит вход по своей почте, код не придёт → выглядит как баг (Guideline 2.1).
3. ⚠️ **Метаданные в App Store Connect** (App Privacy, скриншоты, возрастной рейтинг, заметки ревьюеру) — заполняются при отправке.

Если закрыть эти три — шансы на аппрув высокие.

## ⚠️ Поправить ДО отправки
1. **Email-вход для ревьюера (Guideline 2.1 — самое важное).**
   Resend без верифиц. домена шлёт код **только** на owner-email (`v925619714@gmail.com`).
   Ревьюер своей почтой код не получит → риск пометки «баг».
   **Решение (любое из):** (а) верифицировать домен в Resend (DNS) — тогда коды идут всем;
   (б) в App Review Notes написать: «Sign-in is optional — play as Guest. Multiplayer falls
   back to a bot within ~7s, no second device needed.» (гостевой режим полноценный → этого
   обычно достаточно).
2. **App Privacy («этикетка данных») в App Store Connect** (метаданные, не код): задекларировать
   - *Email Address* — Linked to identity, цель: App Functionality;
   - *User ID* — Linked, App Functionality;
   - *Usage Data / Product Interaction* (PostHog) — **Not Linked**, цель Analytics, **не** Tracking.
3. **Возрастной рейтинг** в ASC: контента 17+ нет → 4+ (или 9+, если анкета поднимет из-за
   онлайн-взаимодействия). «Unrestricted Web Access» = НЕТ.
4. **Скриншоты + описание**: правдивые, реальный геймплей (iPhone 6.7"/6.5" + iPad, т.к. `supportsTablet:true`).
5. **App Review Notes**: как играть (гость, бот-фолбэк), про необязательный вход.

## 🔶 Рекомендуемая доводка (риск ниже, но лучше закрыть)
- **iPad-раскладка.** `supportsTablet:true` → ревью протестируют на iPad. Либо проверить
  верстку поля/HUD на iPad-пропорциях, **либо** выставить `supportsTablet:false`, чтобы не
  ловить iPad-специфичные отказы (для телефонной игры это нормально).
- **Полное удаление auth-записи.** Сейчас `delete_my_account` стирает строку профиля + выход;
  сама запись в `auth.users` остаётся (осиротевшая, анонимная). Данные пользователя удаляются
  → обычно проходит, но для строгого 5.1.1(v) добавить админ-Edge-Function, которая удаляет и
  auth-пользователя.
- **4.3 (клоны/спам).** «Змейка» — классика; снижаем риск тем, что это **соревновательный
  онлайн 1v1 color-duel** + рейтинг/ELO + экономика + редизайн. Риск умеренно-низкий, но Apple
  иногда придирается к казуальным аркадам — помогает уникальная подача в скриншотах/описании.
- **Дальтоник-режим** (red/blue — основа механики дуэли). Не блокер, но повышает доступность
  (и снижает шанс придирки по Accessibility).

## ⛔ Блокеры за пределами кода (нужны действия Владимира)
- **T17 — Apple Developer Program** ($99/год, его не-российская карта). Без активного аккаунта:
  нельзя `eas build --profile production`, нельзя `eas submit`, нельзя завести App-запись/IAP.
- После активации: подписать Free Apps Agreement, создать ключ App Store Connect API (Admin)
  → прислать (Issuer ID, Key ID, `.p8`) → дальше сборку/submit/TestFlight гоню я.
- **Проверка на реальном iPhone** (краши/касания/вибро) возможна только через TestFlight —
  тоже после активации аккаунта.

prod-билд через EAS → TestFlight (тест на iPhone) → submit.
