# Задача: анализ и подробный план развития игры Shake Work Off

Ты — опытный продакт-менеджер + гейм-дизайнер + техлид мобильных игр. Полностью изучи ВЕСЬ проект в этой директории: код, игровую механику, структуру, конфиги, зависимости.

## Что за приложение
**Shake Work Off** (бывш. Chroma Coil) — казуальная игра-змейка на Expo / React Native (TypeScript).
- Есть корпоративный режим **Office Royale** — реалтайм-мультиплеер на 5–10 игроков, «последний выживший сегодня не работает».
- Есть режим **Versus** и одиночная игра с ботами.
- Веб-версия: snake.skillmake.ru. iOS: на ревью App Store (билд 5). Android — в планах.
- Аудитория: офисные работники, антистресс/развлечение в перерыв. Тема — «стряхнуть с себя лишнюю работу».
- Есть аналитика PostHog, неоновый премиум-дизайн (Neon Arena), боты, лидерборд.

## Что нужно сделать
Составь **ПОДРОБНЕЙШИЙ, практичный план развития и правок** по разделам:

1. **Игровая механика и фичи** — что улучшить в геймплее, новые режимы/механики, прогрессия, кастомизация, реиграбельность, глубина Office Royale.
2. **Техническое** — качество кода, рефакторинг, оптимизация производительности (особенно рендер игры + мультиплеер-нетуокинг), архитектура, тех-долг, стабильность реалтайма, тестирование, кроссплатформа (iOS/Android/web).
3. **Монетизация** — какие модели подойдут под офисную аудиторию (IAP, косметика/скины, реклама, подписка, командные/корпоративные тарифы), конкретные предложения с обоснованием.
4. **Удержание (retention)** — онбординг, ежедневные механики, стрики, push-уведомления, социальные фичи, ключевые метрики.
5. **Рост и виральность** — реферальные петли, шеринг результатов, корпоративный англ (команды/офисы/челленджи), ASO для App Store, каналы привлечения.
6. **UX/UI и доступность** — что улучшить в интерфейсе и потоках.
7. **Приоритизация** — разбей всё на этапы: quick wins → средние → крупные. Для каждого пункта оцени impact/effort (низкий/средний/высокий) и предложи порядок.

## Требования к результату
- Для каждого пункта давай **конкретику**: какие файлы/компоненты трогать, примеры, обоснование «почему».
- Пиши развёрнуто, как настоящий план для команды, а не общими словами.
- Опирайся на РЕАЛЬНЫЙ код проекта (сначала изучи его), а не на догадки.

**Запиши весь результат в файл `KIMI-ANALYSIS.md` в этой директории** (markdown, чёткая структура с заголовками).
