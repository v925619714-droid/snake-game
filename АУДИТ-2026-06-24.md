# Chroma Coil — аудит (продукт + техника), 2026-06-24

Провели два специалиста: продукт-аналитик (по продукту/живой версии) и тех-тестировщик (по реальному коду + Supabase). Ниже синтез + план. Источник запроса — упор на **сессии/авторизацию/кросс-девайс**.

## Идентичность СЕЙЧАС (факт по коду)
- Логина нет. Профиль = случайный id устройства (`p_...`, имя `Player-XXXX`, rating/wins/losses) в AsyncStorage (`snake:profile`); кошелёк `snake:wallet`; рекорд `snake:best`.
- Supabase используется только анонимно (publishable key): Realtime (мультиплеер) + таблица `profiles` (лидерборд) через RPC `upsert_profile`. `persistSession:false`.
- **Кросс-девайса нет:** каждое устройство/браузер = новый профиль. Очистка кэша / переустановка / другой браузер = прогресс потерян безвозвратно. Дубли в лидерборде от одного человека.

## P0 — критично до релиза
1. **Подделка рейтинга/лидерборда.** Рейтинг считается на клиенте; RPC `upsert_profile` открыт для `anon` (`grant ... to anon`) без валидации → любой пишет произвольный рейтинг (999999) на любой id, перезаписывает чужие строки. Файлы: `supabase/profiles.sql`, `src/screens/DuelGame.tsx:98-108`, `src/lib/leaderboard.ts`.
2. **Обрыв связи в матче не обрабатывается** — соперник вышел → второй игрок зависает навсегда. `src/net/useRoom.ts` (нет реакции на presence-leave/disconnect).
3. **Рассинхрон мультиплеера** — Supabase Broadcast без гарантий доставки/порядка → у гостя рывки/телепорты (результат матча консистентен, считает хост). Нет sequence-номера/ресинка. `useRoom.ts:109-117, 302-312`.
4. **Поиск без сети зависает** в Quick/Friend (нет таймаута; в Ranked спасает бот-фолбэк).

## P1 — важно
- Гонки в matchmaking при 3+ одновременных игроках (возможны «два хоста»/осиротевший). `useRoom.ts:159-300`.
- Соло-цикл пересоздаёт `setInterval` на каждое съедение → рывок таймингов. `App.tsx:145-149`.
- Гость без `oppRating` → ELO считается «сам против себя». Противоречие: **рейтинг меняется от матчей с ботом** (хотя бот «в лидерборде не участвует»).
- Нет тестов на сеть/профиль (useRoom, profile.ts — 0 тестов).
- Монеты/кошелёк полностью клиентские (правятся в DevTools) — некритично, пока скины косметические.

## Чего не хватает (продукт)
- **Онбординг/туториал** — игрок попадает в игру без объяснения (особенно дуэль) → бьёт по D1.
- **Удержание/лайв-опс** — нет дейликов, наград за вход, сезонов рейтинга, баттл-пасса.
- **Монетизация** — только скины за внутр. монеты; нет баттл-пасса, rewarded-ads, премиум-скинов, IAP-конфига.
- **Социалка/виральность** — нет реферралки, шеринга результата, «вызвать друга».
- **Сторовые требования** — нет privacy policy, нет удаления аккаунта (Apple требует при аккаунтах), не заполнен App Privacy (PostHog), не настроен IAP.
- Мелочи: нет звуков, паузы, дальтоник-режима (red/blue); на iOS нечем поделиться комнатой (нет Share/deep-link); тени на каждой клетке — стоимость на web.

## Рекомендуемая архитектура аккаунтов (Expo + Supabase)
Уйти от device-id к **Supabase Auth (`auth.uid()`)** как единому ключу прогресса; локально — кэш.
1. **Анонимный вход** при первом запуске (`signInAnonymously`) + `persistSession:true` (на iOS токен в SecureStore) → прогресс в облаке, не теряется при очистке кэша.
2. **Апгрейд до аккаунта** без потери прогресса (link identity): **Sign in with Apple** (обязателен на iOS при наличии др. логинов), **Google**, **email magic-link/OTP** (web/Android).
3. **Кросс-девайс**: один `auth.uid()` → один профиль; rating/coins/skins/best синхронятся. «Начал в браузере → продолжил на айфоне» работает.
4. **Сервер считает рейтинг** (Edge Function `report_match`, оба клиента подтверждают исход) + RLS `auth.uid() = id`, запись `rating` с клиента закрыть → чинит накрутку. Бот-матч не репортится.
5. **Слияние дублей** лидерборда при первом логине.
6. **UX:** не просить логин на старте (отвал ~70%) — после 1–3 матчей / при покупке / при попадании в топ: «Сохрани прогресс». Гостевой режим остаётся.

Фазы: **Ф0** срочный стоп-гап на запись рейтинга → **Ф1** анонимный Auth + профиль на `uid` + RLS → **Ф2** Apple/Google/magic-link + апгрейд → **Ф3** серверный ELO + merge дублей → **Ф4** удаление аккаунта (App Store).

## Порядок работ (приступаем)
1. **P0 стоп-гап безопасности** — ограничить накрутку рейтинга в RPC (диапазон + шаг ±64, wins/losses монотонно). Быстро, без пересборки.
2. **Аккаунты + кросс-девайс** (Ф1→Ф2) — главный запрос пользователя.
3. **Устойчивость мультиплеера** (обрыв/таймауты/ресинк).
4. **Онбординг-туториал**.
5. Удержание/лайв-опс, монетизация, сторовая готовность.
