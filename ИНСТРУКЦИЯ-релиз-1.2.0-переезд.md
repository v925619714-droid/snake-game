## 2. Как теперь работает (архитектура)

Всё на 37427 в docker-compose (`/opt/snake-backend/`, SSH: `ssh -i C:\ssh\nl-designer root@151.245.137.127`):

| Что | Технология | Адрес |
|---|---|---|
| Данные (лидерборд/профили/кошелёк) | Postgres 15 + PostgREST | `https://snake-api.skillmake.ru/rest/v1` |

- **Анти-чит** серверный (submit_match: рейтинг соперника из БД, кулдаун, дневной кап) — работает, проверено.
- **119 профилей** перенесены (лидерборд не пустой).

## 3. Что изменилось в коде (для ревью)

- `src/net/useRoom.ts`, `src/net/usePartyRoom.ts` — переписаны с `supabase.channel` (Supabase Realtime) на **`colyseus.js`** (наш сервер). Интерфейсы хуков сохранены → экраны (DuelGame/PartyGame) не менялись. Убралось ~942 строки старого peer-host кода.
- `src/lib/appConfig.ts` + гейт в `App.tsx` — экран «обновите приложение», если версия < `min_version` из таблицы `app_config` на бэкенде. Сейчас `min_version=1.0.0` → **неактивен** (никого не блокирует). Для будущих cutover'ов: поднять min_version → старые версии попросят обновиться.
- `metro.config.js` — **важный фикс**: `colyseus.js` тянет `ws`/`httpie` (Node-модули `stream`/`https`, которых нет в React Native). Направил Metro на browser-условие (colyseus.js/httpie резолвятся в свои browser-сборки на глобальном WebSocket + XMLHttpRequest, которые в RN есть). Без этого native-бандл падает на фазе «Bundle JavaScript».
- `eas.json` — добавлен профиль `production-apk` (APK для RuStore; `production` собирает AAB).
- `app.json` — версия `1.1.0` → `1.2.0`.
- **Сервер (не в git, на 37427):** Caddy получил CORS-preflight для gateway (иначе supabase-js из браузера падал `Failed to fetch`); таблица `app_config`; бэкап-таймер. Все грабли задокументированы в `ПЛАН-переезда-бэкенд.md`.
- **EAS production env** переведён на новый бэкенд (`EXPO_PUBLIC_SUPABASE_URL`→snake-api, новый anon-ключ, добавлен `EXPO_PUBLIC_GAME_WS`=snake-rt). POSTHOG без изменений.

## 5. Что делает КОМП 1 (пошагово)

Всё из папки `snake-game`. Node в PATH. `EXPO_TOKEN` — в SECRETS (аккаунт kanaewvs-team, Owner).

### A. iOS — собрать и залить (нужен вход в Apple)
```powershell
git pull                      # забрать релиз 1.2.0
# ИНТЕРАКТИВНО (валидирует сертификат, попросит войти в Apple):
npx eas-cli@latest build --platform ios --profile production
# ↑ на вопрос про credentials — согласиться, войти в Apple ID разработчика.
# После сборки — отправка в App Store (ключ ASC — в SECRETS, .p8 положить в snake-game/asc-api-key.p8):
npx eas-cli@latest submit --platform ios --profile production --latest
```
В App Store Connect: привязать билд к версии 1.2.0, release notes, отправить на ревью. (Домен apple может быть заблокирован в браузере — см. `appstore-automation-limits`.)

### B. Android — залить APK в RuStore
1. Дождаться `finished` сборки `a55d2901` (или `npx eas-cli@latest build:list --platform android --limit 1`), скачать `.apk` (Application Archive URL).
2. В RuStore Console — загрузить APK как обновление (versionCode 6 > текущего 4). Проверить манифест (RECORD_AUDIO/SYSTEM_ALERT_WINDOW/STORAGE заблокированы в app.json — RuStore это требует).
3. (Google Play — только если аккаунт уже одобрен: собрать `production` (AAB) и залить.)
